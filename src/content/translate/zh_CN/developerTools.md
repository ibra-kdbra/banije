## 🛠️ Web 开发工具

 作为一名 Web 应用工程师，我不断寻找能够提高我的技能和生产力的工具。 以下是基本工具的更新列表，其中包含 2024 年和 2025 年的最新进展。 

---

 ## 现有工具

 ### 🛠️ Wappalyzer

:::important
[Purpose]
识别网站上使用的技术。
:::**为什么使用它**：帮助识别 Web 服务器、CMS、框架等。 它非常适合从其他网站学习并发现可能适合您自己的项目的技术。 

---

 ### 🎨 手写笔

:::important
[Purpose]
自定义网站外观。
:::**为什么使用它**：在任何网站上自定义 CSS 样式。 对于试验设计和修复字体大小或配色方案等布局问题很有用。 

---

 ### 🧑‍💻 JSON 视图

:::important
[Purpose]
格式化并验证 JSON 数据。
:::**为什么使用它**：帮助读取和理解来自 API 的 JSON 数据。 整齐地格式化数据并帮助调试语法或逻辑错误。 

---

 ### 📜 HTML 验证器

:::important
[Purpose]
检查 HTML 代码是否有错误。
:::**为什么使用它**：确保 HTML 代码有效并遵循标准。 有助于提高代码质量和网站性能。 

---

 ### 📏 测量一下

:::important
[Purpose]
测量网页上的元素。
:::**为什么使用它**：允许精确测量网站上的图像、文本、按钮和其他元素。 对于优化布局和设计很有用。 

---

 ### 🌐 REST 客户端

:::important
[Purpose]
测试和调试 API。
:::**为什么使用它**：允许您向任何 API 发送 HTTP 请求、查看响应并分析状态代码和数据，从而帮助进行 API 测试和调试。 

---

 ### 🎨 ColorZilla

:::important
[Purpose]
选择并分析颜色。
:::**为什么使用它**：帮助为您的网站选择颜色并分析其他网站的配色方案。 非常适合创建一致、有吸引力的设计。 

---

 ### 🧪 硒 IDE

:::important
[Purpose]
自动化网络测试。
:::**为什么使用它**：在浏览器中记录和回放自动化测试。 帮助确保您的网站正常运行并节省手动测试时间。 

---

 ### 🛠️ 网页开发人员

:::important
[Purpose]
操纵和检查网站。
:::**为什么使用它**：提供禁用 JavaScript、检查 cookie、查看源代码、验证链接等功能，这些功能对于调试和增强网站都至关重要。 

---

 ### 🔤WhatFont

:::important
[Purpose]
识别网页上的字体。
:::**为什么使用它**：快速识别网站上使用的字体并显示字体属性。 对于为您自己的项目选择和应用正确的字体很有用。 

---

 ### 🌍WhatRuns

:::important
[Purpose]
发现网站使用的技术。
:::**为什么使用它**：识别任何网站上使用的插件、小部件、分析和框架。 帮助跟踪趋势并发现改进您自己的项目的工具。 

---

 ## 适用于前端、后端和全栈开发人员的其他工具

 ---

 ### 📱 React 开发者工具

:::important
[Purpose]
调试 React 应用程序。
:::**为什么使用它**：对于调试 React 应用程序至关重要，允许您检查组件层次结构、状态和属性，帮助优化用户界面。 

---

 ### 🖥️ Redux 开发工具

:::important
[Purpose]
调试 Redux 状态更改。
:::**为什么使用它**：跟踪 Redux 支持的应用程序中的操作和状态变化。 非常适合使用 React 和 Redux 的全栈开发人员。 

---

 ### 🧑‍💻 Vue.js 开发工具

:::important
[Purpose]
调试 Vue.js 应用程序。
:::**为什么使用它**：提供用于检查 Vue 组件、Vuex 状态等的工具，对于使用 Vue.js 的前端开发人员来说是不可或缺的。 

---

 ### 🐍 邮递员

:::important
[Purpose]
API开发和测试。
:::**为什么使用它**：可以轻松地向 API 发送请求、查看响应和测试 API 端点。 对于后端开发人员或使用 REST API 的任何人来说都是必不可少的。 

---

 ### 🖧 实时服务器

:::important
[Purpose]
启动本地开发服务器。
:::**为什么使用它**：立即启动本地服务器并进行实时重新加载，非常适合在前端开发期间需要快速反馈的全栈开发人员。 

---

 ### 🌐 GitHub

:::important
[Purpose]
增强 GitHub 工作流程。
:::**为什么使用它**：直接从浏览器提供通知、拉取请求和问题等功能，使协作和管理存储库变得更加容易。 

---

 ### ⚙️ Gitpod

:::important
[Purpose]
在云中启动开发环境。
:::**为什么使用它**：为全栈开发人员提供基于云的 IDE，让您可以轻松地从任何设备进行编码并与 GitHub 存储库同步。 

---

 ### 📑 JSON 格式化程序

:::important
[Purpose]
格式化和美化JSON数据。
:::**为什么使用它**：自动格式化 JSON 数据，使其更具可读性且更易于调试。 对于使用 API 的后端开发人员特别有用。 

---

 ### 🖥️失眠

:::important
[Purpose]
API设计和测试。
:::**为什么使用它**：用于测试 RESTful API 和 GraphQL 的强大工具。 它支持调试、请求和响应，并允许创建复杂的 API 工作流程。 它是使用 API 驱动的应用程序的开发人员的必备工具。 

---

 ＃＃＃ GitLab

:::important
[Purpose]
在代码存储库上进行协作。
:::**为什么使用它**：基于 Git 的版本控制的流行平台，允许开发人员管理项目、跟踪问题以及与团队协作。 GitLab 非常适合开源和私有项目。 

---

 ### 🔧 代码博格

:::important
[Purpose]
Git 存储库托管。
:::**为什么使用它**：Codeberg 是 GitHub 的社区驱动替代品，提供了一个更加开源的平台来托管 Git 存储库。 对于想要替代主流服务的开发人员来说，这是一个不错的选择。 

---
