## 🛠️ Web 开发工具

作为一名 Web 应用工程师，我一直在寻找能提升技能和生产力的工具。以下是一份更新后的必备工具清单，结合了 2024 年和 2025 年的最新进展。

---

## 现有工具

### 🛠️ Wappalyzer

:::important[用途]
识别网站使用的技术栈。
:::

**使用理由**：有助于识别网站的 Web 服务器、CMS、框架等。这对于借鉴其他网站的经验以及发现适合自己项目的技术非常有帮助。

---

### 🎨 Stylus

:::important[用途]
自定义网站外观。
:::

**使用理由**：自定义任何网站的 CSS 样式。非常适合进行设计实验，或修复字体大小、配色方案等布局问题。

---

### 🧑‍💻 JSON View

:::important[用途]
格式化并校验 JSON 数据。
:::

**使用理由**：有助于阅读和理解来自 API 的 JSON 数据。它可以整齐地格式化数据，并帮助调试语法或逻辑错误。

---

### 📜 HTML Validator

:::important[用途]
检查 HTML 代码错误。
:::

**使用理由**：确保 HTML 代码有效并符合标准。有助于提高代码质量和网站性能。

---

### 📏 Measure-it

:::important[用途]
测量网页上的元素。
:::

**使用理由**：允许精确测量网页上图片、文字、按钮和其他元素的尺寸。有助于优化布局和设计。

---

### 🌐 REST Client

:::important[用途]
测试和调试 API。
:::

**使用理由**：允许你向任何 API 发送 HTTP 请求、查看响应并分析状态码和数据，从而辅助 API 的测试与调试。

---

### 🎨 ColorZilla

:::important[用途]
提取并分析颜色。
:::

**使用理由**：有助于为网站选择配色，并分析其他网站的配色方案。对于创建统一且美观的设计非常有帮助。

---

### 🧪 Selenium IDE

:::important[用途]
自动化 Web 测试。
:::

**使用理由**：在浏览器中录制和回放自动化测试。有助于确保网站功能正常，并节省手动测试的时间。

---

### 🛠️ Web Developer

:::important[用途]
操作和检查网站。
:::

**使用理由**：提供了禁用 JavaScript、检查 Cookie、查看源代码、验证链接等功能，这些对于调试和增强网站至关重要。

---

### 🔤 WhatFont

:::important[用途]
识别网页字体。
:::

**使用理由**：快速识别网站使用的字体并显示属性。对于在自己的项目中选择和应用合适的字体非常有用。

---

### 🌍 WhatRuns

:::important[用途]
发现网站使用的技术。
:::

**使用理由**：识别任何网站使用的插件、小部件、分析工具和框架。有助于跟踪趋势并发现能改进自己项目的工具。

---

## 面向前端、后端及全栈开发者的进阶工具

---

### 📱 React Developer Tools

:::important[用途]
调试 React 应用。
:::

**使用理由**：调试 React 应用的必备工具，允许你检查组件层级、State 和 Props，从而优化用户界面。

---

### 🖥️ Redux DevTools

:::important[用途]
调试 Redux 状态变更。
:::

**使用理由**：跟踪 Redux 应用中的 Action 和 State 变化。非常适合使用 React 和 Redux 的全栈开发者。

---

### 🧑‍💻 Vue.js Devtools

:::important[用途]
调试 Vue.js 应用。
:::

**使用理由**：提供检查 Vue 组件、Vuex 状态等功能，是使用 Vue.js 的前端开发者的必备工具。

---

### 🐍 Postman

:::important[用途]
API 开发与测试。
:::

**使用理由**：轻松发送 API 请求、查看响应并测试 API 端点。对于后端开发者或任何使用 REST API 的人来说必不可少。

---

### 🖧 Live Server

:::important[用途]
启动本地开发服务器。
:::

**使用理由**：立即启动带有实时重载功能的本地服务器，非常适合需要前端开发快速反馈的全栈开发者。

---

### 🌐 GitHub

:::important[用途]
增强 GitHub 工作流。
:::

**使用理由**：直接在浏览器中提供通知、Pull Request 和 Issue 管理功能，使协作和仓库管理变得更加容易。

---

### ⚙️ Gitpod

:::important[用途]
在云端启动开发环境。
:::

**使用理由**：为全栈开发者提供基于云的 IDE，让你能够轻松地从任何设备进行编码，并与 GitHub 仓库同步。

---

### 📑 JSON Formatter

:::important[用途]
格式化和美化 JSON 数据。
:::

**使用理由**：自动格式化 JSON 数据，使其更具可读性且易于调试。对于处理 API 的后端开发者特别有用。

---

### 🖥️ Insomnia

:::important[用途]
API 设计与测试。
:::

**使用理由**：一款测试 RESTful API 和 GraphQL 的强大工具。它支持调试、请求与响应，并允许创建复杂的 API 工作流。对于处理 API 驱动型应用的开发者而言是必备品。

---

### 🌍 GitLab

:::important[用途]
代码仓库协作。
:::

**使用理由**：一个流行的基于 Git 的版本控制平台，允许开发者管理项目、跟踪问题并与团队协作。GitLab 非常适合开源和私有项目。

---

### 🔧 Codeberg

:::important[用途]
Git 仓库托管。
:::

**使用理由**：作为 GitHub 的社区驱动替代方案，Codeberg 提供了一个更专注于开源的 Git 仓库托管平台。对于希望寻找主流服务替代品的开发者来说，这是一个很好的选择。

---
